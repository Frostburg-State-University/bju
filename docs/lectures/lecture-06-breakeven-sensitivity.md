# Lecture 6 — Break-even & Sensitivity Analysis

!!! info "Session Info"
    **Date:** Monday, 2026-06-08 &nbsp;·&nbsp; **Time:** 16:20–18:10 &nbsp;·&nbsp; **Room:** YF108
    **Instructor:** Dr. Zhijiang Chen

## Learning Objectives

1. Compute **break-even points** in units, dollars, and time.
2. Perform **one-variable** and **multi-variable sensitivity analysis** on an NPV model.
3. Construct and interpret **Tornado diagrams**.
4. Identify which inputs most strongly drive a software-project decision.

## Topics

- Break-even analysis for software products: usage, subscribers, throughput.
- Sensitivity tables and spider plots.
- Tornado diagrams: ranking input drivers.
- "What-if" thinking as a discipline, not a feature of Excel.

## In-Class Exercise

For a hypothetical AI feature with $200K dev cost, $0.02 marginal cost per request, and projected adoption, find the request-volume break-even and produce a Tornado diagram across five inputs.

## Homework

- Build a sensitivity workbook for one project. Identify the top three NPV drivers.
