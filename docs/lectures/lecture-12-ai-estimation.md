## Learning Objectives

1. Explain why **LOC and Function Points lose accuracy** with AI-assisted development.
2. Critique the "lines per day" metric in the era of code generation.
3. Read recent empirical data on AI coding-tool productivity (mixed effects across seniority).
4. Propose alternative productivity measures suited to AI-augmented teams.

## Topics

- The 2026 productivity paradox: junior developers +10–30%, senior developers −19% (verification overhead).
- Why size-based estimation fragments when code is largely AI-authored.
- Outcome-based metrics: tickets resolved, features shipped, defect-free deployment frequency.
- Calibrating COCOMO II for AI-assisted projects.
